# Logbook Protocol

Two files; pick the right one.

- **`STATUS.md`** — durable handbook (auto-loaded by CLAUDE.md).
  Current state, sentinel addresses, how-to-test pointers, model
  gotchas, open work. Edit when one of those facts changes.
- **`LOGBOOK.md`** — chronological work entries, newest first. Append
  when you finish (or significantly progress) a piece of work.

Concise. No session-by-session journals. Capture what a future agent
needs to know — the finding, the file paths, the verification step,
the gotcha — and move on.

If your entry would change a fact in STATUS.md (a new sentinel
address, a fresh gotcha, a change in current state), update STATUS.md
in the same commit.
